# ot_compliance_pack

Assembles an ISA/IEC-62443 compliance evidence bundle for OT environments. The role collects audit events, generates role review and rule-diff reports, and compresses all artifacts into a dated ZIP archive suitable for submission to auditors or compliance teams.

## Requirements

- Ansible 2.12+
- Write access to `artifacts_dir` on the control host
- `zip` utility available on the Ansible control host (for bundle creation)
- Audit event data pre-populated in `audit_events` variable or generated by upstream tasks

## Role Variables

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `artifacts_dir` | `/tmp/ot-artifacts` | No | Directory where individual evidence files and the final ZIP bundle are written. |
| `audit_events` | `[]` | No | List of audit event records to include in the compliance bundle. Typically populated by upstream inventory or alert-collection tasks that feed data from OT security platforms (Claroty, Dragos, etc.). Each record should include at minimum: `timestamp`, `event_type`, `asset`, `user`, and `description`. |

## Tasks Executed

The role runs four include-task phases in order:

1. **pull_audit** — Formats and writes the `audit_events` list to a structured CSV evidence file.
2. **role_reviews** — Generates a role and privilege review report listing users, roles, and last-access timestamps from the audit event stream.
3. **rule_diffs** — Produces a rule-diff report comparing current firewall/segmentation rules against the approved baseline for ISA/IEC-62443 zone-conduit requirements.
4. **bundle_zip** — Compresses all artifact files in `artifacts_dir` into a dated ZIP archive named `ot_compliance_bundle_<date>.zip`.

## Dependencies

None. Upstream tasks that populate `audit_events` (such as `ot_alerts__pull_normalize.yml` or OT platform inventory tasks) should be run before this role.

## Example Playbook

```yaml
---
- name: Generate OT ISA-62443 compliance bundle
  hosts: localhost
  connection: local
  gather_facts: false

  vars:
    artifacts_dir: "/opt/ot/compliance/{{ lookup('pipe','date +%Y%m%d') }}"
    audit_events: "{{ ot_audit_data }}"  # populated by upstream tasks

  pre_tasks:
    - name: Pull OT audit events from platform
      ansible.builtin.include_tasks: operational_technology/tasks/ot_alerts__pull_normalize.yml

    - name: Pull OT inventory for rule review
      ansible.builtin.include_tasks: operational_technology/tasks/ot_inventory__discover_from_platform.yml

  roles:
    - role: operational_technology/roles/ot_compliance_pack
```

## Output Artifacts

- `{{ artifacts_dir }}/audit_events.csv` — Formatted audit event evidence.
- `{{ artifacts_dir }}/role_reviews.csv` — User and privilege review report.
- `{{ artifacts_dir }}/rule_diffs.csv` — Firewall/segmentation rule comparison against approved baseline.
- `{{ artifacts_dir }}/ot_compliance_bundle_<date>.zip` — Compressed ZIP archive of all evidence files for audit submission.

## Compliance Mapping

This role supports evidence collection for the following ISA/IEC-62443 requirement areas:

- **SR 1.1** — Human user identification and authentication (role reviews)
- **SR 2.1** — Authorization enforcement (rule diffs)
- **SR 2.8 / SR 2.9** — Auditable events and audit log protection (audit events)
- **SR 3.2** — Malicious code protection (alert events)
